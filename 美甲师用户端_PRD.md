## 一、产品定位

### 1.1 产品定义

面向**美甲师私域客户**的轻量移动端 WebApp，核心场景：

- 通过美甲师邀请链接 / 邀请码完成注册并绑定美甲师
- 浏览美甲师作品集，互动（点赞 / 收藏 / 评论）
- 发起预约、提交设计需求
- 与美甲师实时沟通
- 管理上门服务地址与订单

### 1.2 用户获取机制（已实现）

```
美甲师分享邀请链接（/invite?code=xxx）
→ 用户点击进入
→ 手机号 + 验证码注册，填写昵称
→ 自动绑定该美甲师
```

补充规则：
- 从无邀请码入口进入时，注册流程要求手动填写邀请码
- 注册后首次进入强制跳转 Welcome 页完成昵称设置
- 已注册用户可在【我的】页面继续绑定新的美甲师
- 可设置默认美甲师，所有首页 / 作品 / 消息数据围绕默认美甲师展示

### 1.3 核心约束

| 约束 | 状态 |
|------|------|
| 不做平台分发，不做美甲师搜索 | ✅ |
| 用户只能通过邀请码绑定美甲师 | ✅ |
| 用户可绑定多个美甲师，可设置默认 | ✅ |
| 不内置线上支付，仅记录定金状态 | ✅ |

## 二、核心业务流程

```
进入邀请链接
→ 手机号验证码注册 + 填写昵称
→ 自动绑定美甲师
→ 首页浏览作品
→ 发起设计需求（上传参考图）或 直接新建预约
→ 等待美甲师报价
→ 确认报价 → 线下支付定金 → 标记已付定金
→ 美甲师确认订单
→ 服务完成
```

## 五、数据模型（关键表）

| 表名 | 用途 |
|------|------|
| ClientUser | 客户账号（手机、昵称、头像、状态）|
| ClientTechBinding | 客户与美甲师绑定关系（isDefault、status）|
| ClientAddress | 客户地址（多地址、经纬度）|
| Order | 预约订单（报价、定金、状态机）|
| ClientDesignRequest | 设计需求（图片、报价、状态）|
| CustomServiceRequest | 自定义服务请求（服务类型、地址、日期）|
| Conversation + Message | 会话与消息（文本 / 图片 / 系统消息）|
| NailWork | 作品（可见 / 置顶 / 精品标记）|
| NailWorkComment | 评论（树形回复、置顶 / 隐藏 / 软删）|
| NailWorkLike / NailWorkFavorite | 点赞与收藏记录 |
