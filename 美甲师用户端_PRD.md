

# 💅 美甲客户用户端 PRD（WebApp｜绑定美甲师私域工具）**

------

# **一、产品定位**

## **1.1 产品定义**

本产品是一个基于 WebApp 的轻量客户端，用于：

👉 **服务“美甲师私域客户”**

核心作用：

- 客户与美甲师建立连接
- 提供预约与沟通能力
- 提供款式浏览与设计能力
- 支持上门服务信息管理

------

## **1.2 用户来源（关键机制）**

用户获取方式：

```
美甲师分享名片（链接 / 二维码）
→ 用户点击进入
→ 注册账号
→ 自动绑定该美甲师
```

补充规则：

- 用户从非邀请链接进入时，注册前必须填写美甲师提供的邀请码
- 首次注册时必须引导用户填写姓名
- 已注册用户可在【我的】页面继续绑定新的美甲师

------

## **1.3 核心约束（必须统一）**

### **❌ 不做**

- 不做平台分发
- 不做技师搜索

### **✅ 必须**

- 用户只能通过邀请码与美甲师建立绑定关系
- 用户可绑定多个美甲师
- 预约时可从已绑定的美甲师中选择目标美甲师
- 美甲师可解除与客户的绑定关系
- 首页、作品、消息、预约等数据必须明确关联具体美甲师

------

# **二、核心业务流程（最重要）**

```
进入邀请链接 / 输入邀请码 → 注册 → 绑定美甲师
→ 浏览作品 / 沟通
→ 发起设计 or 预约
→ 美甲师报价
→ 用户确认 + 支付定金（线下）
→ 美甲师确认订单
→ 上门服务
→ 完成
```

------

# **四、核心模块设计**

------

# **4.1 首页（Home）**

## **功能目标**

建立信任 + 快速转化预约

------

## **模块1：美甲师名片**

字段：

- 美甲师头像
- 名字
- 简介
- 服务区域
- 联系方式（可选）

------

## **模块2：作品展示**

功能：

- 图片瀑布流
- 点击查看详情
- 详情页显示该作品所属美甲师名称
- 支持收藏（可选）

------

## **模块3：快捷操作**

- 预约美甲
- 发起设计
- 联系美甲师

------

# **4.2 预约模块（Booking）**

------

## **4.2.1 新建预约**

字段：

| **字段** | **类型** |
| -------- | -------- |
| 美甲师   | string   |
| 时间     | datetime |
| 地址     | string   |
| 服务类型 | string   |
| 备注     | text     |

规则：

- 预约时必须选择一个已绑定的美甲师
- 若用户没有地址，则先引导新增地址后再提交预约
- 若用户已有地址，则可从地址列表中选择
- 用户可维护多个地址信息

------

## **4.2.2 预约状态流（重要）**

```
待报价 → 已报价 → 待确认 → 已预约 → 已完成 → 已取消
```

------

## **4.2.3 我的预约列表**

字段：

- 美甲师名称
- 时间
- 状态
- 报价金额
- 地址

------

# **七、关键设计原则（非常重要）**

------

## **1️⃣ 极简路径（转化优先）**

- 首页 → 一键预约
- 首页 → 一键发设计

------

## **2️⃣ 强绑定（私域）**

- 用户只属于一个美甲师
- 所有内容围绕该美甲师

------

## **3️⃣ 轻交易（避免复杂支付）**

- 不内置支付
- 仅做状态记录

------

## **4️⃣ 信任优先**

- 强化作品展示
- 强化真实感（而非平台感）
