# 美甲师客户端_PRD

## 一、项目概述

### 1.1 产品定位

本产品是一款面向**独立上门美甲师**的移动端 WebApp，用于管理客户、订单及行程，提高接单效率与服务质量。

核心目标：
- 提升接单效率
- 优化行程安排
- 增强客户复购率



### 1.2 目标用户

| 用户类型   | 特征             |
| ---------- | ---------------- |
| 自由美甲师 | 无门店，上门服务 |
| 兼职美甲师 | 非全职接单       |
| 小团队     | 1-3人合作        |



### 1.3 使用场景

- 手机端为主（iOS / Android）
- WebApp（PWA形式）

## 三、功能模块说明

### 3.1 首页（Dashboard）

### 功能目标
快速掌握当天工作情况

---

### 模块1：今日行程
字段：
- 时间（14:00 - 16:00）
- 客户姓名
- 地址（可导航）
- 状态（待出发 / 服务中 / 已完成）

交互：
- 点击进入订单详情
- 支持导航跳转

---

### 模块2：今日收入
- 今日总收入
- 已完成订单金额

---

### 模块3：待处理订单
- 待确认订单
- 待报价订单

---

### 模块4：快捷操作
- 新建订单
- 新建客户
- 查看行程

### 3.2 行程（Schedule）

---

### 3.2.1 日历视图

支持：
- 日视图（默认）
- 周视图

数据结构：

```json
{
  "date": "2026-04-28",
  "orders": [
    {
      "id": "ord_001",
      "start_time": "14:00",
      "end_time": "16:00",
      "customer_name": "小美",
      "status": "confirmed"
    }
  ]
}
```



**3.2.2 冲突检测**

规则：

```
新订单.start_time < 已有订单.end_time
AND
新订单.end_time > 已有订单.start_time
→ 判定冲突
```

表现：

- 红色提示
- 弹窗提醒

------

### **3.2.3 地图路线**

功能：

- 展示当天订单位置
- 按时间排序路线（MVP）

------

### **3.2.4 订单状态流**

```
待确认 → 已确认 → 出发 → 服务中 → 已完成
```

## **五、接口设计（API）**

### **获取订单列表**

```
GET /orders?date=2026-04-28
```



### **创建订单**

```
POST /orders
```



### **更新订单状态**

```
PATCH /orders/{id}/status
```



### **获取客户列表**

```
GET /customers
```



### **新建客户**

```
POST /customers
```



## **六、核心流程**

### **创建订单流程**

```
首页 → 新建订单 → 选择客户 → 填写信息 → 保存
```



### **工作流程**

```
首页 → 行程 → 订单 → 出发 → 服务 → 完成
```



### **客户复购**

```
客户详情 → 新建订单
```
