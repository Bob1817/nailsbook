## 一、产品定位

### 1.1 产品定义

面向**独立上门美甲师**的移动端 WebApp，用于管理客户、订单、行程、作品集和店铺信息，提升接单效率与私域运营能力。

核心目标：
- 提升接单效率（一键确认报价、状态流转）
- 优化行程安排（周视图日历、行程列表）
- 增强客户复购率（客户档案、标签、消息触达）
- 建立作品展示与私域引流能力

### 1.2 使用场景

- 手机端（iOS / Android）WebApp 为主
- 所有页面已完成 mobile-first 响应式设计，最小触控区 44px

### 1.3 入驻方式

```
提交美甲师申请（姓名/手机/城市/服务方式/经验）
→ 超管后台审核
→ 审核通过后生成邀请 Key
→ 美甲师使用邀请 Key 完成注册
```

## 四、数据模型（关键表）

| 表名 | 用途 |
|------|------|
| Technician | 美甲师账号（手机、密码、头像、服务配置）|
| TechnicianInviteKey | 注册邀请 Key |
| Customer | 客户档案（标签、备注、偏好）|
| Order | 订单（含状态机、报价、定金、冲突检测）|
| NailWork | 作品（可见 / 置顶 / 精品 / 图片 / 标签）|
| NailWorkComment | 评论（树形、置顶 / 隐藏 / 软删）|
| TechnicianService | 服务项目（名称、价格、类型、状态）|
| TechnicianSubscription | 订阅状态（计划、有效期）|
| Conversation + Message | 会话与消息 |
| Revenue | 收入记录（关联已完成订单）|

## 五、API 接口（前缀 `/api/v1/technician/`，需 Bearer Token）

| 模块 | 典型接口 |
|------|---------|
| 认证 | POST /auth/login · POST /auth/register · GET /auth/me · PATCH /auth/profile |
| 订单 | GET /orders · POST /orders · PATCH /orders/:id/review · PATCH /orders/:id/confirm |
| 客户 | GET /customers · GET /customers/:id · PATCH /customers/:id/tags |
| 作品 | GET /works · POST /works · PATCH /works/:id · POST /works/:id/toggle-featured |
| 评论 | GET /works/:id/comments · POST /works/:id/comments · POST /works/:workId/comments/:commentId/pin |
| 服务 | GET /services · POST /services · PATCH /services/:id/toggle |
| 消息 | GET /messages/conversations · POST /messages（+ WebSocket）|
