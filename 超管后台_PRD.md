# **💼 超级管理后台 PRD（最终版｜SaaS工具）**

------

# **一、项目概述**

## **1.1 产品定位**

本系统为：

👉 **面向独立美甲师的预约与客户管理 SaaS 系统的后台管理平台**

核心职责：

- 管理美甲师（B端用户）
- 管理报价与预约数据
- 管理订阅与收费
- 提供数据分析能力
- 控制系统规则

------

## **1.2 产品本质**

| **类型**     | **说明**     |
| ------------ | ------------ |
| ❌ 交易平台   | 不参与支付   |
| ❌ 抽佣平台   | 无分账       |
| ✅ SaaS工具   | 按订阅收费   |
| ✅ Booking OS | 以预约为核心 |

------

## **1.3 核心业务闭环**

```text
客户沟通 → 报价 → 支付定金（线下） → 美甲师确认 → 形成预约 → 服务完成 → 自动计入收入
```

# **四、核心业务模型（必须严格实现）**

------

## **4.1 报价（Quote）**

```json
{
  "id": "quote_001",
  "customer_id": "cust_001",
  "tech_id": "tech_001",
  "price": 300,
  "deposit": 50,
  "status": "pending | accepted | expired",
  "created_at": ""
}
```

------

## **4.2 预约（Booking）**

```json
{
  "id": "booking_001",
  "quote_id": "quote_001",
  "start_time": "",
  "end_time": "",
  "status": "pending_confirm | confirmed | completed | cancelled",
  "is_deposit_paid": true
}
```

------

## **4.3 收入（Revenue）**

```json
{
  "id": "rev_001",
  "booking_id": "booking_001",
  "amount": 300,
  "status": "confirmed"
}
```

------

# **五、核心状态流（必须统一）**

------

## **5.1 报价**

```text
pending → accepted → 生成预约 → expired
```

------

## **5.2 预约**

```text
待确认（已付定金）
→ 已确认
→ 服务中（可选）
→ 已完成
→ 已取消
```

------

## **5.3 收入规则（核心逻辑）**

```text
预约完成 → 自动生成收入
收入 = 报价金额
```

# **八、关键规则（必须严格执行）**

------

## **1️⃣ 定金规则**

```text
只记录是否支付，不做支付系统
```

------

## **2️⃣ 收入规则**

```text
收入必须来源于报价
禁止手动录入
```

------

## **3️⃣ 数据可信性**

- 报价不可随意修改
- 完成后不可改价

------

## **4️⃣ 完成机制**

- 支持自动完成
- 支持手动完成
