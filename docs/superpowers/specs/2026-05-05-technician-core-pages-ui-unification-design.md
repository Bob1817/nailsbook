# 美甲师端五个核心页面 UI 统一改造设计

- 日期：2026-05-05
- 范围：美甲师端 `首页 / 行程 / 客户 / 消息 / 我`
- 目标：在不改变核心业务结构的前提下，统一五个核心页面的视觉语言、布局密度和移动端阅读体验

## 1. 设计结论

本次改造采用以下统一方向：

1. 首页保留情绪化和品牌感
2. 行程、客户、消息、我四页统一为克制的信息页风格
3. 四个信息页以白卡片为主，局部用浅粉标签、主按钮、选中态和重点数字提气色
4. 页面节奏采用折中方案：
   - 首屏保证核心信息完整
   - 卡片之间保留适度呼吸感
   - 不为了“展示更多”牺牲清晰度

## 2. 现状问题

当前五页的主要问题不是单页视觉失衡，而是跨页面不一致：

1. 首页和“我”页采用强渐变头图和上浮统计卡，氛围感强，但细节精度不完全一致
2. 行程页已经偏向信息页，但地图区、统计卡、时间轴的密度与其他页不统一
3. 客户页和消息页更接近功能列表页，但标题区、筛选区、卡片区的层级略散
4. 五页中的卡片圆角、阴影强度、标题字号、按钮重量、标签尺寸存在差异
5. 部分页面首屏信息过满，部分页面又留白过虚，整体节奏不稳定

## 3. 总体设计原则

### 3.1 视觉角色分工

五页不再追求“同一张脸”，而是做明确分工：

- 首页：品牌感、状态感、收入感
- 行程：信息效率、时间与路线组织
- 客户：客户列表与关系管理
- 消息：提醒优先级与聊天入口
- 我：个人工作台与配置入口

首页承担情绪化入口，其他四页承担信息效率。

### 3.2 统一视觉语言

信息页统一遵守以下规则：

1. 页面背景采用浅粉白底，不再使用大面积高饱和渐变
2. 卡片统一白底、大圆角、轻阴影
3. 浅粉色仅用于：
   - 主按钮
   - 当前选中态
   - 标签
   - 强调数字
   - 关键操作入口
4. 文字层级统一为：
   - 页面标题：最强
   - 模块标题：次强
   - 列表主信息：清晰但不过重
   - 次信息：浅灰
5. 不使用过重边框和过强阴影

### 3.3 布局密度规则

四个信息页统一采用折中密度：

1. 首屏必须能看到核心信息和首个主模块
2. 每个模块之间保持稳定呼吸感
3. 同一张卡片内信息优先分层，不强求全部横向塞入一行
4. 小屏优先避免横向挤压和换行变形
5. 内容不足一屏时不强制造成页面滚动感，内容超过一屏时自然滚动

## 4. 页面级设计

### 4.1 首页

首页保留现有情绪化结构，但做精致化收口。

保留：

1. 顶部渐变头图
2. 美甲师头像、名称、接单状态开关
3. 分享名片入口
4. 上浮统计卡

调整：

1. 顶部信息区减少无意义装饰光斑，突出状态与收入信息
2. 头图中的左右信息区层级更清楚：
   - 第一层：身份与接单状态
   - 第二层：今日工作摘要
   - 第三层：收入信息
3. 上浮统计卡继续和头图重叠，但阴影和上浮幅度更柔和
4. 下方“今日行程”“快捷操作”等模块采用信息页白卡风格，与内页视觉语言接上

首页最终效果应当像“品牌首页 + 工作摘要”，而不是纯后台面板。

### 4.2 行程页

行程页作为最典型的信息页，需要进一步克制和统一。

保留：

1. 顶部标题与日期条
2. 统计卡
3. 地图/列表双模式
4. 今日行程时间轴
5. 定金提醒卡

调整：

1. 日期条按钮更统一，选中态更轻，辅助红点更克制
2. 统计卡的四列文字、数字、单位全部统一层级，避免局部字号失衡
3. 地图区弱化背景噪声，让模式切换、节点卡、路线按钮成为真正视觉重点
4. 时间轴卡片减少视觉负担，提升时间、客户名、服务标签、金额的阅读顺序
5. 按钮统一为轻描边次按钮和浅粉主按钮两类

行程页最终效果应当像“日程信息面板”，而不是带大量装饰的功能页。

### 4.3 客户页

客户页要强调筛选效率和客户关系管理。

调整方向：

1. 顶部标题区变轻，不使用情绪化大头图
2. 搜索框和筛选 tab 放在统一白卡或轻底容器中
3. 客户卡统一为：
   - 左侧头像
   - 中间客户主信息与标签
   - 右侧最近服务或消费信息
4. 详情抽屉或详情区也延续白卡与浅粉标签语言
5. “查看订单 / 新建预约 / 联系客户”等动作按钮统一重量

客户页最终效果应当更像“客户工作台”，而不是社交列表页。

### 4.4 消息页

消息页需要把“聊天”和“业务提醒”区分清楚，但不做成两个完全割裂的产品。

调整方向：

1. 标题区和右上角操作区更简洁
2. 顶部筛选 tab 统一成浅底胶囊式切换
3. 聊天对话卡与业务提醒卡共用基础卡片规则，但通过标签和文案层级做区分
4. 未读点、提醒数、状态色统一，不再出现多个红点样式
5. 系统通知、客户提醒、服务提醒的右侧操作入口视觉统一

消息页最终效果应当像“业务消息中心”，而不是混杂的通知流。

### 4.5 我

“我”页保留个人工作台属性，但从视觉上向信息页靠拢，不再像第二张首页。

保留：

1. 顶部渐变信息区
2. 上浮统计卡
3. 收入统计
4. 工具区
5. 设置区

调整：

1. 顶部渐变区保留，但弱于首页，避免两页抢角色
2. 统计卡和收入卡保持白卡统一风格
3. 工具区与设置区减少杂色块，统一为白卡内的轻色图标底板
4. 信息分组更明确：
   - 数据
   - 工具
   - 邀请
   - 设置

“我”页最终效果应当像“个人运营控制台”，而不是营销式展示页。

## 5. 统一组件规则

本次改造会尽量复用已有组件，并统一以下规则：

### 5.1 卡片

1. 信息页主卡统一白底
2. 统一大圆角
3. 统一轻阴影
4. 不再按页面各自定义不同卡片重量

### 5.2 标签

1. 状态标签统一胶囊样式
2. 浅粉用于主强调
3. 浅橙用于提醒
4. 浅绿用于完成或已收
5. 中性色用于普通状态

### 5.3 按钮

按钮只保留三类：

1. 主按钮：浅粉或粉红渐变，承载核心操作
2. 次按钮：浅底、轻描边
3. 文本按钮：用于“查看全部”“查看订单”等轻操作

### 5.4 标题区

四个信息页统一改成轻标题区：

1. 页面标题明显
2. 副标题更短
3. 右侧操作只保留关键入口

## 6. 响应式与移动端约束

本次改造必须继续遵守移动端优先：

1. 不允许核心区块出现左右滑动
2. 不允许依靠裁切掩盖溢出
3. 小屏下优先缩字、分层、换布局，不强塞一行
4. 所有按钮和可点击区保持触控友好
5. 固定底部导航存在时，卡片和弹层要考虑安全区与底部避让

## 7. 范围控制

本次只做 UI 和布局统一，不扩新功能，不改业务流程。

明确不包含：

1. 新增后端接口
2. 改变订单、客户、消息的核心业务逻辑
3. 重做导航结构
4. 新增复杂动效系统

允许的范围：

1. 页面结构微调
2. 视觉层级重排
3. 组件样式统一
4. 小范围文案层级调整

## 8. 验收标准

完成后应满足：

1. 首页保留品牌感，但不与“我”页重复抢角色
2. 行程、客户、消息、我四页一眼看出属于同一套克制信息页体系
3. 五页卡片、标签、按钮、标题层级统一
4. 小屏宽度下不出现横向滑动和明显排版挤压
5. 首屏核心信息完整，滚动节奏自然
6. 不改变现有业务闭环与主操作路径

## 9. 实施顺序

推荐实施顺序：

1. 先统一基础卡片、标签、按钮、标题区规则
2. 再改首页和“我”页，先把头图区和上浮统计卡统一关系收顺
3. 再改行程页，收口最复杂的信息结构
4. 最后改客户页和消息页，统一列表卡片和筛选区

这样可以减少反复返工。
