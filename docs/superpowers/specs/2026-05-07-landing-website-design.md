# NailBook 官网需求与设计文档

## 一、背景与目标

### 1.1 背景

`landing-frontend` 是 NailBook 的独立官网项目，当前已具备官网首页与角色选择页雏形，但存在两个直接问题：

- 现有官网样式在运行态未正确生效，需要修复。
- 官网信息架构过轻，无法承接品牌展示、角色分流、入驻转化与轻业务展示。

本次工作需要先完成官网需求与设计定义，再进入实现计划与开发。

### 1.2 产品目标

NailBook 官网本次承担四个明确目标：

1. 建立统一品牌入口，向用户清楚说明平台价值。
2. 承接登录入口，先做角色分流，再跳转到对应业务端登录页。
3. 承接美甲师入驻转化，提供轻量申请入口。
4. 通过公开作品/服务展示增强平台可信度，但不在本轮实现完整交易市场。

### 1.3 核心转化优先级

- 主 CTA：`立即登录 / 选择角色`
- 次级 CTA：`美甲师入驻`
- 辅助 CTA：`查看作品/服务`

### 1.4 目标人群

官网面对双边人群，但首页叙事优先顺序如下：

1. 美甲师与潜在入驻美甲师
2. 美甲客户

首页必须体现双边价值，但视觉与文案重心优先服务美甲师转化，客户价值作为辅助证明与次级入口呈现。

## 二、范围定义

### 2.1 本轮范围

本轮只覆盖以下前端范围：

- `landing-frontend` 官网项目
- `technician-frontend` 美甲师登录页的角色切换入口
- `client-frontend` 客户登录/注册页的角色切换入口

### 2.2 本轮包含内容

- 修复官网样式不生效问题
- 重构官网首页信息架构与 UI
- 保留并优化角色选择页
- 新增美甲师入驻页
- 新增作品/服务展示页
- 在美甲师登录页增加切换到客户端登录页的入口
- 在客户端登录/注册页增加切换到美甲师登录页的入口

### 2.3 本轮不包含内容

- 不接真实的入驻审核后台
- 不接真实的官网 CMS
- 不接完整作品 marketplace 或搜索筛选系统
- 不重构客户端或美甲师端整体视觉系统
- 不新增后台管理功能

## 三、需求文档

### 3.1 官网首页 `/`

#### 目标

用一个移动端优先的官网首页同时完成品牌介绍、双边价值说明、主次 CTA 引导和公开内容展示。

#### 功能要求

首页必须包含以下模块：

1. Hero 首屏
2. 双角色价值概览
3. 平台核心功能说明
4. 使用流程说明
5. 作品/服务展示预览
6. 美甲师入驻 CTA 区块
7. FAQ 或常见问题
8. Footer

#### 交互要求

- 主按钮点击后进入 `/role-select`
- 次级按钮点击后进入 `/artist-join`
- 辅助内容可引导到 `/showcase`
- 页面内导航支持滚动定位
- 移动端所有关键点击区域不小于 `44px`

### 3.2 角色选择页 `/role-select`

#### 目标

在用户正式登录前先做角色确认，降低误入错误业务端的概率。

#### 功能要求

- 展示两个角色卡片：
  - `我是美甲师`
  - `我是美甲客户`
- 页面保留返回官网入口
- 每张卡片有简短角色说明

#### 跳转要求

- 选择美甲师：跳转 `http://localhost:5175/login`
- 选择美甲客户：跳转 `http://localhost:5173/login`

### 3.3 美甲师入驻页 `/artist-join`

#### 目标

承接次级转化，让有兴趣的美甲师能提交入驻意向。

#### 表单字段

- 姓名
- 手机号
- 所在城市
- 服务方式
- 从业年限
- 擅长风格
- 补充说明

#### 交互要求

- 表单提交后展示成功状态
- 本轮可以是纯前端提交完成态，不依赖真实 API
- 页面应保留返回首页与登录入口

### 3.4 作品/服务展示页 `/showcase`

#### 目标

用公开内容增强平台可信度与专业度，证明客户体验和美甲师作品能力。

#### 内容要求

- 精选作品卡片
- 风格标签或服务标签
- 服务流程或预约体验说明
- 登录与入驻两个 CTA

#### 边界

- 本轮不做真实筛选、下单或搜索闭环
- 页面定位是内容展示页，不是完整 marketplace

### 3.5 美甲师登录页角色切换

#### 目标

让进入错误登录页的用户能低成本纠正路径。

#### 功能要求

- 在美甲师登录页新增次级文本入口：
  - `切换角色：我是美甲客户`
- 点击后跳转 `http://localhost:5173/login`

#### 位置要求

- 放在标题区附近或表单底部
- 不得抢占主登录按钮层级

### 3.6 客户登录/注册页角色切换

#### 目标

在客户端登录和新用户注册场景下都允许用户切换到美甲师端。

#### 功能要求

- 新增次级文本入口：
  - `切换角色：我是美甲师`
- 点击后跳转 `http://localhost:5175/login`

#### 位置要求

- 在登录态与注册态下都可见
- 视觉上保持次级，不干扰验证码登录/注册主操作

## 四、设计文档

### 4.1 信息架构

本轮官网采用以下路由结构：

- `/`：首页
- `/role-select`：角色选择页
- `/artist-join`：美甲师入驻页
- `/showcase`：作品/服务展示页

首页作为统一品牌入口，次级内容页只承接某一类明确任务，不再把所有内容堆在一个长页面里。

### 4.2 首页模块顺序

首页建议采用以下内容顺序：

1. Hero：突出美甲师经营与客户体验的双边价值，主叙事偏美甲师
2. 角色入口概览：明确平台服务两类用户
3. 核心功能：预约、客户管理、作品展示、沟通、收入分析
4. 平台如何运作：用 3 到 4 步说明流程
5. 作品/服务展示预览：增强真实感和可信度
6. 入驻申请 CTA：承接美甲师转化
7. FAQ：处理典型疑问
8. Footer：站点导航与协议说明

### 4.3 视觉方向

视觉方向采用：`柔和轻奢编辑感`

具体原则如下：

- 不采用强科技感 SaaS 风格
- 以奶油白、暖粉、浅玫瑰、深墨色为主色系统
- 页面使用柔和渐变、光晕色块和分层阴影，避免纯白平铺
- 标题有更强视觉张力，正文保持高可读性
- 卡片、按钮、表单输入均采用大圆角与明确按压反馈

### 4.4 移动端优先约束

官网与登录相关页面必须满足以下移动端约束：

- 布局先按手机纵向浏览设计，再向大屏扩展
- 核心点击区域高度不低于 `44px`
- 固定头部、浮层与页脚按钮需考虑安全区
- 不使用 hover 作为唯一交互提示
- 表单、卡片与按钮在小屏上保持足够留白与清晰层级

### 4.5 角色选择页设计

角色选择页使用两张大卡片垂直排布为主，桌面端可提升为双列。

设计要求：

- 每张卡片有角色图标、标题、说明、子标签
- 操作反馈明确，按压后立即跳转
- 页面保留 `返回官网` 文本或箭头入口

### 4.6 入驻页设计

入驻页定位为轻量转化表单页，风格延续官网，但信息密度高于首页。

设计要求：

- 表单主体放在单独卡片中
- 顶部用简要文案说明平台对美甲师的价值
- 提交成功态要清晰独立，不与表单混淆

### 4.7 Showcase 页设计

Showcase 页定位为内容证明页。

设计要求：

- 卡片化展示精选作品或风格
- 使用标签、统计或说明块增强结构化阅读
- 页面底部或中段可再次承接登录与入驻 CTA

### 4.8 登录页角色切换设计

两个业务端登录页中的角色切换入口都采用次级文本按钮形式，不引入新的主按钮，不改变既有登录流程。

设计原则：

- 文案明确说明“切换角色”
- 保持弱于登录主操作的视觉层级
- 与现有页面风格兼容，不做大面积重构

## 五、技术设计

### 5.1 已知现状

`landing-frontend` 当前技术栈为：

- React
- TypeScript
- Vite
- React Router
- Tailwind CSS 4
- Lucide React

项目已可成功构建并输出 CSS 文件，因此样式问题更可能发生在运行态接入、Tailwind 4 使用方式或具体样式组织上，而不是构建失败。

### 5.2 实现策略

官网实现遵循以下策略：

1. 先修复样式失效问题，确保全局样式和 Tailwind 工具类在开发态稳定生效
2. 在不引入不必要抽象的前提下重构首页
3. 新增少量页面满足转化闭环
4. 登录页只做与角色切换直接相关的最小改动

### 5.3 跳转策略

- 官网内部页面：使用 `react-router-dom`
- 跨前端项目跳转：继续使用显式 URL 跳转
- 角色选择页作为统一登录前分流页，不在首页直接写死跳转到某一业务端

### 5.4 数据策略

本轮不接真实后端数据时，可采用静态内容或本地表单完成态。

约束如下：

- 不伪造复杂接口层
- 不为了一个表单新增全局状态管理
- 若后续接 API，应在不推翻路由和信息架构的前提下扩展

## 六、验收标准

### 6.1 功能验收

- `landing-frontend` 样式恢复正常生效
- 首页、角色选择页、入驻页、showcase 页均可访问
- 首页主 CTA 点击后进入角色选择页
- 首页次级 CTA 可进入美甲师入驻页
- 角色选择页能正确分流到两个业务端登录页
- 美甲师登录页可切换到客户端登录页
- 客户登录/注册页可切换到美甲师登录页

### 6.2 设计验收

- 官网整体视觉明显优于当前版本
- 移动端浏览优先，交互控件触控友好
- 信息层级清晰，不出现双主线抢占首屏的问题
- 首页叙事偏美甲师，但客户价值仍然可理解

### 6.3 变更边界验收

- 仅修改与本次目标直接相关的页面与样式
- 不引入未经要求的业务能力
- 不对客户端或美甲师端做无关重构

## 七、风险与注意事项

- 现有官网项目规模小，若一次性组件化过度，容易引入不必要复杂度，应保持页面级直接实现优先。
- 两个业务端登录页结构不同，角色切换入口需要分别适配现有布局，但应保持统一文案逻辑。
- 若后续需要真实入驻提交接口，本轮前端表单结构应尽量与常见 API 载荷兼容，避免返工。

## 八、实施前结论

本次方案采用“美甲师优先的双边平台官网”：

- 官网首页负责品牌表达与主转化
- 角色选择页负责登录前分流
- 入驻页负责美甲师次级转化
- Showcase 页负责公开内容证明
- 两个业务端登录页负责角色纠错入口

该方案在当前代码基础上改动可控，满足需求闭环，同时保留后续对真实表单提交、公开内容数据化和 SEO 扩展的空间。
