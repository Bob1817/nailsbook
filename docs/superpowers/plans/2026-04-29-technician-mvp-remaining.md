# Technician MVP Remaining Task List

**Goal:** 完成美甲师客户端 MVP 业务闭环，让美甲师可以登录后完成预约创建、预约处理、客户联动和基础提醒。

## 已完成基础

- 登录页与 technician 鉴权已接通
- 首页、行程、客户、消息、我的基础页已存在
- technician 侧客户列表、客户详情、预约列表、预约详情读取已接通
- 预约确认 / 完成 / 取消接口已存在
- 订单页表单与本地草稿流已存在，但未接入真实创建

## 剩余 MVP 任务

### P0：预约创建闭环

- [ ] 后端新增 `POST /api/technician/bookings`
- [ ] 创建预约时自动生成关联报价单，满足现有 `Booking.quoteId` 约束
- [ ] 校验客户归属必须属于当前 technician
- [ ] 前端订单页提交改为真实创建接口
- [ ] 创建成功后在订单页、行程页、首页、客户详情中可见
- [ ] 仅当接口失败时才退回本地草稿

### P0：预约页正式接入

- [ ] `/orders` 路由不再跳转 `/schedule`
- [ ] 底部导航“订单/行程”与实际页面职责统一
- [ ] 订单页创建提示文案从“本地草稿”改成“创建预约”

### P1：预约状态流完整化

- [ ] 前端订单页支持待确认 -> 已确认
- [ ] 前端订单页支持已确认 -> 已完成
- [ ] 前端订单页支持取消预约
- [ ] 状态变更后刷新订单、首页、行程、消息页联动数据

### P1：客户详情联动

- [ ] 客户详情直接发起新预约
- [ ] 客户详情展示预约历史与最近服务信息

### P1：消息提醒闭环

- [ ] 待确认预约提醒
- [ ] 即将上门提醒
- [ ] 定金未收提醒
- [ ] 预约状态变化提醒

### P2：可用性补强

- [ ] technician 端关键表单做基础校验与冲突校验
- [ ] 关键创建/状态操作增加明确成功与失败反馈
- [ ] 主要页面继续按统一 UI token 收口
